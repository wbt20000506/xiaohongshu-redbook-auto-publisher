# 小红书卡片图片生成器

将 Markdown 文本转换为小红书风格的高清卡片图片，并生成配套的发帖文案。

## 完整工作流程

用户给出一个主题/话题后，按以下步骤操作：

### 第一步：生成文案

根据 `references/copywriting-guide.md` 中的写作指南，为用户生成小红书发帖文案。

文案包含三部分，写入 `copy.md`：
- **标题选项**（3 个备选，遵循标题公式）
- **正文**（发帖正文，不是卡片内容，语言口语化、有共鸣、反 AI 味）
- **标签**（15-20 个，包含大流量 + 精准 + 长尾）

> 注意：文案正文是贴在小红书帖子里的文字，和卡片图片上的内容**可以不同**。
> 卡片是视觉内容（干货、要点、排版），文案是帖子描述（引导阅读、互动、共鸣）。

### 第二步：准备卡片内容

根据主题，规划卡片图片上要展示的内容（干货要点、结构化信息等）。
如果内容较多，用 `---` 分隔成多页。

### 第三步：预览风格（打开浏览器让用户看）

```bash
node scripts/generate-image.js \
  --title "卡片标题" \
  --content "卡片上的内容" \
  --preview
```

浏览器会打开展示 10 种风格缩略图。**不保存任何文件**。等待用户选择。

### 第四步：正式生成（用户选好风格后）

```bash
node scripts/generate-image.js \
  --title "卡片标题" \
  --content "卡片上的内容" \
  --style <风格ID>
```

生成高清卡片 + copy.md，保存到：
`/Users/wave/Documents/xhs-card/<YYYYMMDD-HHmmss-标题>/`

### 第五步：补充 copy.md

将第一步生成的文案内容写入输出目录的 `copy.md` 文件。

## 参数说明

- `-t, --title <title>`: 卡片标题（必填）
- `-c, --content <content>`: 卡片图片上的内容（必填）
- `-o, --output <name>`: 输出目录名后缀（可选，默认用标题）
- `-s, --style <id>`: 指定风格 ID 进行正式生成
- `--preview`: 在浏览器中打开预览，不保存文件

```

├── copy.md          # 发帖文案（标题选项 + 正文 + 标签）
├── slide-01.png     # 第 1 页卡片
├── slide-02.png     # 第 2 页卡片
└── ...
```

## copy.md 格式规范

```markdown
# 小红书文案

## 标题选项

1. 标题A
2. 标题B
3. 标题C

## 正文

（发帖正文，口语化，有共鸣，反AI味）

## 标签

#标签1 #标签2 #标签3 ...
```
