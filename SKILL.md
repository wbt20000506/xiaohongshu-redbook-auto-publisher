# 小红书内容创作

## 设计哲学

**「像书页，不像海报」** — 零装饰、强排版、反 AI 味。

核心原则：
- 奶白底 `#f5f0eb`，无渐变
- 强调只有两种：**加粗** + **荧光笔划线**（每页 ≤ 1 处）
- 每页内容密度有下限，杜绝"三行一页"的空洞感
- 无 emoji、无边框、无阴影、无圆角
- 字体层级清晰：标题 → 正文 → 脚注

**严禁出现的元素（反 AI 味铁律）：**
- 无作者名、头像、日期、水印
- 无 "by AI"、"Generated by"、"AI 创建" 等任何 AI 来源标识
- 无页码指示器（如 "1/3"）
- 无 CTA 按钮、无引导文案
- 无任何装饰性元素（渐变条、图标、分隔线）
- 卡片只包含：标题 + 正文内容，干净利落

### 内容格式

用 `-f` 传入的文件支持以下 Markdown 子集：

```markdown
# 大标题（会忽略，用 -t 传入）

## 小标题
正文内容

**加粗文字**
==高亮划线==
```

- `##` 开头的行渲染为小标题（字号 42px，加粗）
- `**文字**` 渲染为加粗
- `==文字==` 渲染为荧光笔划线
- 空行分段
- 其他行渲染为正文

## 一、卡片图片生成

### 风格选择

根据内容类型选择风格，参考 `references/styles.json`：

| 风格 | ID | 适合场景 |
|------|-----|---------|
| 深色科技 | dark-tech | GitHub Trending、AI 工具、技术排名 |
| 奶白书页 | cream-page | 知识分享、教程、笔记（默认） |
| 苹果极简 | apple-minimal | 产品发布、正式公告 |
| 柔和卡片 | pastel | 生活方式、轻松话题 |
| 杂志风 | magazine | 长文分析、行业报告 |

### 输出目录

所有输出统一放在 `~/Documents/xhs-card/` 下，每次生成创建新文件夹：

文件夹命名格式：`{时间戳}-{主题关键词}`

```
~/Documents/xhs-card/
├── 20260611-143052-效率工具/
│   ├── copy.md
│   ├── slide-01.png
│   ├── slide-02.png
│   └── ...
└── ...
```

### 使用流程

当用户要求生成卡片时，先询问风格选择：

> 选择卡片风格：
> A. 奶白书页（知识分享、教程、笔记）
> B. 深色科技（GitHub Trending、AI 工具）
> C. 苹果极简（产品发布、正式公告）
> D. 柔和卡片（生活方式、轻松话题）
> E. 杂志风（长文分析、行业报告）

用户选择后执行：

```bash
OUTPUT_DIR=~/Documents/xhs-card/$(date +%Y%m%d-%H%M%S)-主题关键词
mkdir -p "$OUTPUT_DIR"

# 写内容到临时文件（保留换行符）
cat > /tmp/card-content.txt << 'EOF'
内容第一行
内容第二行

空行分段
EOF

node /Users/wave/shared-skills/xhs-card/scripts/generate-image.js \
  -t "标题" \
  -f /tmp/card-content.txt \
  -s 风格ID \
  -o "$OUTPUT_DIR/slide-01.png"
```

> **重要**：必须用 `-f` 从文件读取内容，不要用 `-c` 直接传字符串。
> `-c` 传参时 shell 会吞掉换行符，导致所有文字挤成一段。

### 参数

| 参数 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| `-t, --title` | ✅ | - | 卡片标题 |
| `-f, --content-file` | ✅ | - | 内容文件路径，`---` 分隔多页 |
| `-c, --content` | ❌ | - | 直接传内容（**不推荐**，shell 会吞换行） |
| `-s, --style` | ❌ | cream-page | 风格 ID |
| `-o, --output` | ❌ | card.png | 输出路径 |

### 输出规格

| 项目 | 值 |
|------|-----|
| 卡片尺寸 | 1080 × 1440（3:4，小红书竖图） |
| 实际像素 | 2160 × 2880（2x DPI） |
| 格式 | PNG |
| 命名 | `slide-01.png` … `slide-0N.png` |

## 二、文案生成

生成完卡片后，主动询问用户：

> 卡片已生成，需要配套文案吗？
> A. 小红书文案（标题选项 + 正文 + 20 个标签）
> B. 微头条文案（严格 150 字）
> C. 不需要

### 小红书文案规范

**标题**（选 3 个备选）：
- 18-20 字，不带 emoji（反 AI 味）
- 使用「数字+关键词+结果」结构
- 示例：`5 个效率工具，让你每天多出 2 小时`

**正文**：
- 300-500 字
- 分段清晰，每段 2-3 行
- 口语化、有共鸣
- 不用 emoji，用标点符号控制节奏
- 结尾加互动引导

**标签**：
- 20 个相关标签
- 格式：`#标签名`

### 保存位置

文案保存到输出目录的 `copy.md` 文件。
